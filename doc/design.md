# Project Magpie (鹊桥 Agent)：异地恋智能出行决策系统

## 1. 项目愿景 (One-Line Pitch)
一款运行在个人服务器上的全自动“量化买票”与度假决策 Agent。它通过持续监控航线与高铁价格，结合时序价格预测模型和多维度外部信息（天气、节假日），在最佳时机自动触发购票提醒，将异地恋的每月出行成本降至最低，体验拉至最满。

## 2. 背景与痛点 (The Problem)
- **空间跨度大**：南昌与北京相隔约 1,400 公里，每月一次的会面（或前往中间城市度假）依赖高铁与飞机，交通成本是核心开销。
- **价格波动黑盒**：机票价格随时间、政策、上座率剧烈波动，人工比价极其消耗精力，且容易错过“抄底”时机。
- **决策维度繁杂**：完美的“休闲度假风（Staycation）”不仅取决于票价，还受目的地天气、酒店价格波动、甚至两人当月日程的影响。人工统筹这些信息极度低效。

## 3. 核心功能 (Core Features)
- **7x24 小时暗中观察**：突破传统 OTA 平台限制，全自动抓取特定航线（如南昌 ⇄ 北京，或至南京、杭州等中间点）的票价与车次余票。
- **票价量化预测（买点探测）**：将金融量化交易的思路“降维打击”到机票购买上。通过构建历史价格时序数据库，建立动态基准线，精准识别“绝对低价”与“下跌趋势”。
- **AI 多维决策大脑**：接入大语言模型，将价格数据与目的地天气预报、周末活动等动态上下文结合。系统不仅告诉你“票便宜了”，更会告诉你“下周末秦皇岛机票跌破 400，且恰逢晴天，极其适合看海，建议买入”。
- **无感触发推送**：达到理想阈值后，通过 Webhook 瞬间将格式化的决策建议推送到手机微信/飞书。

## 4. 系统架构与技术路径 (Architecture)
系统设计为四个高度解耦的模块，便于快速迭代和 Vibe Coding 自动生成：

1. **感知层 (Ingestion)**：基于 Amadeus API 或无头浏览器（Playwright）的高频抓取脚本，伪装请求，获取干净的 JSON 价格流。
2. **记忆层 (Storage & Time-Series)**：使用轻量级本地数据库（SQLite / DuckDB），以时间戳为索引记录每一次抓取的价格快照，构建专属“票价 K 线图”。
3. **决策层 (Agentic Brain)**：
   - *Phase 1 (MVP)*: 简单的移动平均线（MA）策略，跌破均线即报警。
   - *Phase 2*: 引入轻量级时序模型预测未来三天价格走势。结合 LLM 进行自然语言摘要分析。
4. **触达层 (Action/Alert)**：轻量级的通知分发网关，直达用户终端。
