# Project Magpie (鹊桥 Agent)：异地恋智能出行决策系统

## 1. 项目愿景 (One-Line Pitch)
一款运行在个人服务器上的全自动“量化买票”与度假决策 Agent。它通过持续监控航线与高铁价格，结合时序价格预测模型和多维度外部信息（天气、节假日），在最佳时机自动触发购票提醒，将异地恋的每月出行成本降至最低，体验拉至最满。

## 2. 背景与痛点 (The Problem)
- **空间跨度大**：南昌与北京相隔约 1,400 公里，每月一次的会面（或前往中间城市度假）依赖高铁与飞机，交通成本是核心开销。
- **价格波动黑盒**：机票价格随时间、政策、上座率剧烈波动，人工比价极其消耗精力，且容易错过“抄底”时机。
- **决策维度繁杂**：完美的“休闲度假风（Staycation）”不仅取决于票价，还受目的地天气、酒店价格波动、甚至两人当月日程的影响。人工统筹这些信息极度低效。

## 3. 核心功能 (Core Features)
- **极速高频暗中观察**：底层采用 Rust/Tokio 中枢与原生 Python/cffi 强力探针，绕开 WAF、TLS 防线，纯净抓取 12306 与 OTA 的原始 JSON 余票及票价流，做到了资源消耗极低的每秒高频探伤。
- **大盘量化降维打击**：将金融量化交易思路移植。在 SQLite 中建立每一次时间片段的价格快照矩阵，并自动识别断崖式下跌与虚假低价。
- **Agentic 大脑 (DeepSeek V3.2 决策)**：接入最强的 DeepSeek 大模型推理接口，把生冷的数字、当地的暴雨预警和用户的日历进行情感加工。“不仅帮你算好了钱，还帮你安排了约会的晴天。”
- **无感触发推送**：达到理想阈值后，通过 Webhook 瞬间将格式化的决策建议推送到手机微信/飞书。
- **大模型上下文拓展桥 (MCP Ready)**：兼容飞常准官方 Tripmatch MCP (或者将自身逆向获取的 12306 接口包装为 MCP Server)，成为 Cursor 等本地 IDE 能够直接调用的超级出行决策外脑。

## 4. 系统双轨架构与交互生命周期
为了彻底解耦重型的网关调度与容易被封杀的敏捷爬虫，同时实现极致流畅的“咨询到执行”体验，系统采用 **前置意图驱动 + Rust/Python 双引擎** 架构：

### Phase 0: 意图唤醒与配置层 (The Consultation Brain)
- **自然语言接驳**：摒弃传统的“出发地、目的地、日期”死板表单输入。用户通过自然语言描述构想，例如：“我下个月比较闲，想带女朋友去海边度个假，预算两人机票一千以内能不能搞定？”。
- **意图编译 (LLM => Quant Config)**：DeepSeek V3.2 充当“首席旅行规划师”，进行多轮反问与需求收敛。随后，大模型负责将这段感性的构想**编译并降维**为一份精确的 JSON 监控配置文件（包含：容差时间窗、备选目的地矩阵、价格绝对阈值、量化触发频次）。

### Phase 1: 极速感知层 (Ingestion - Python magpie_agent)
- **协议欺诈与抓取尖刀**：例如利用 `curl_cffi` 模拟 `chrome110` 绕开 12306 的 TCP 层 TLS 切断防线。结合飞常准 Tripmatch MCP，获取多维原始数据流。

### Phase 2: 记忆与调度层 (Storage - Rust magpie_core)
- **极简调度心脏**：使用 Tokio 按照 Phase 0 设定的个性化频率（如：悠闲型用户每天查 2 次，商务急客每 1 分钟查一次），定时拉起探针。
- **持久化堡垒与量化指标计算**：SQLite 保存价格快照矩阵。Rust 引擎内置基础数学评估库，计算滑动均值、布林带等，拦截无效噪音。

### Phase 3: 决断与触达层 (Action & Alert)
- 当量化引擎抛出“异常低价”或“余票告急”的事件探针后，再次唤醒 DeepSeek (Phase 0 大脑)，加载实时天气、个人日历等。最终由 LLM 拍板是否推送提醒，实现精准的情绪价值触达。
