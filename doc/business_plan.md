# Magpie 商业化闭环与演进计划 (Business Plan)

## 1. 核心商业假设与价值锚点
我们的核心不仅仅是一个冷冰冰的比价脚本，而是将**时间成本、情绪价值与信息差**进行变现。本系统最宽广的护城河在于：**量化级别的价格捕捉速度** + **大模型多维决策（天气/日历情感连接）** + **私密化部署。**

## 2. 交互范式颠覆：从“填表”到“前置对话咨询”
我们创造性地用 LLM **前置吸收用户模糊意图**，并将其编译为后台量化参数，针对不同用户画像提供截然不同的**介入与干预周期**：

### A. 时间自由的极限捡漏客（弹性度假与学生群体）
- *模糊意图*：“接下来三个月随时可以走，只要去西南地区吃美食，总价最低就行。”
- *系统干预机制*：极宽的监控窗口（季度级别）。每天仅进行极低频探测（避免消耗服务器资源）。只有总量化模型判断价格跌入历史 5% 绝对底部，或命中极低尾单时，系统才会突然“诈尸式”提醒。
- *商业模式*：“一次性买断低价漏网之鱼推送费”，对价格敏感度极高，重单次成单提成。

### B. 商务差旅刚需群 / 节假日必须见面的异地恋（时间定死群体）
- *强制意图*：“这周五下班前必须到北京，不论什么方式，只求性价比或抢到票。”
- *系统干预机制*：极窄的监控窗口（以小时计）。在起飞或发车前 72 小时，进入“高频狂暴探测模式”（例如每分钟轮询）。运用量化预测分析剩余席位消耗速率，当模型判断“现在不买绝对会涨/卖空”时，立刻触发警报让用户买入。
- *商业模式*：按月度/年度的高订阅费（如 **￥99/月** 或 B端包年）。购买的是绝对的“确定性”和免去人工焦虑的情绪价值。

## 3. 从独立开发者 (Indie Hacker) 到 SaaS 的实施链路
**Phase 1: 社区化与流量漏斗 (Building in Public)**
- **行动**：在 Github 开源以获得极客社区的青睐。利用 PR 和反馈快速迭代 Magpie Core 的反爬机制并打磨 Agent 提示词。
- **预期**：建立初始用户信任背书。

**Phase 2: 降维形态 (DaaS 托管版)**
- **行动**：普通情侣或非技术用户不想经历租服务器、部署 Rust 环境和应对反爬虫的折磨。我们将推出云端代运行服务。
- **预期**：用户通过极其轻量的 Web UI 配置微信推送、绑定私人 ics 日历。后端统一在强大的反爬隧道代理下运行。转入订阅制 MRR (每月经常性收入) 阶段。

**Phase 3: 桌面产品形态 (Tauri Native Client)**
- **行动**：使用 Tauri 将强悍的 Rust `magpie_core` 包裹入轻量的桌面 App。
- **预期**：开箱即用的跨系统超级应用，允许深度技术玩家买断终身授权版本。
